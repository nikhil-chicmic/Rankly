# Implementation Plan: Light/Dark Theme System

## Overview

Install Redux Toolkit, create the light theme and Redux slice, wire the Provider into App.tsx, refactor every component that currently imports `colors` statically to use `useAppTheme()` + `createStyles(theme)`, add the theme toggle row to the Profile screen's Settings card, and write property-based and unit tests for all seven correctness properties.

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP
- Each task references specific requirements for traceability
- Checkpoints ensure incremental validation
- Property tests validate universal correctness properties defined in the design document
- Unit tests validate specific examples and edge cases
- The existing `src/theme/color.ts` must never be modified — all new files import from it read-only
- Shared style-sheet files (e.g. `resume.styles.ts`, `Interview.styles.ts`) are converted to factory functions in task 7; their callers are updated in tasks 14–15
- `ErrorBoundary` is a class component — wrap it in a functional component to access `useAppTheme()` (task 8.13)
